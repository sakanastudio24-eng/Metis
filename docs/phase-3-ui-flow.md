# Phase 3 UI Flow

Phase 3 should make the panel feel useful, not just technical.

## Full Panel Layout

Top block:

- score
- severity label
- one-line cost-risk summary

Issue block:

- 3 to 5 issue cards
- title
- severity
- one-sentence explanation

Breakdown block:

- score deductions
- single-page vs multipage toggle
- original vs current compare

Offender block:

- top offenders
- large images

## Mini Panel Layout

Keep it tighter:

- score
- top 2 issues
- expand button

## Copy Style

Avoid raw-only language such as:

- "124 requests"

Prefer:

- "High request count may increase bandwidth costs"
- "Duplicate requests suggest redundant loading"
- "Large images are driving a meaningful share of page weight"

## Visual Rule

Phase 3 should visually promote:

- score first
- issues second
- diagnostics third

The technical metrics should support trust, not dominate the panel.
