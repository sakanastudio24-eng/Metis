Metis Capability Lab

One page to stress the full Phase 4 shell.

Use the links below to switch scenarios. Each scenario changes the route and loads a different resource pattern so you can test Metis loading gates, issue ordering, stack detection, and the refined report sections.

Scenario Notes

Baseline

A controlled route with enough activity to open Metis, but without deliberately noisy fetch or image behavior.

Expected Stack

Some modes intentionally leave parts of the stack missing so the Improve Accuracy flow can ask for the missing context.

AI-like Interaction

Type to simulate expensive keystroke-driven work.

Each input event triggers a local request and, in stack-present mode, also attempts a no-cors request against an AI-style endpoint so the route feels expensive in motion.

Heavy Media

Oversized image stress area

This mode loads multiple large remote images so Metis can surface media weight and bandwidth pressure.

Content Surface
Route navigation

Switching scenario links changes window.location.href, which is the main trigger you want for the Metis loading replay.

Duplicate requests

Duplicate and AI modes hit the same JSON endpoint with different query params so Metis can normalize them back to one path.

Third-party + stack

Stack-present mode pulls in recognizable framework, analytics, payment, CDN, and AI-style endpoints. Stack-missing mode strips them out on purpose.
